▣ 개발자의 원칙은 중복을 제거하는 것이다. 
▣ 웹팩(webpack) : 어러 개의 자바스크립트 파일을 하나의 자바스크립트 파일로 합치는 것을 의미함.
▣ 웹팩(webpack)을 만들려면 노드(node.js)를 알아야 함. 참고로 노드(node.js)는 자바스크립트 실행기이다.
▣ 웹팩(webpack)을 설치하는 방법.
   - npm init -> package.json 파일 생성
   - npm i react react-dom -> react, react-dom 설치
   - npm i -D webpack webpack-cli -> webpack, webpack-cli 설치, -D : 개발용
   - package.json 파일에 있는 폴더에 webpack.config.js 파일과 client.jsx 파일 생성
▣ "dependencies" : 실제 서비스에 사용되는 플러그인 목록, "devDependencies" : 개발에 사용되는 플러그인 목록
▣ 확장자가 jsx 파일이면 js 파일과 구별되므로 개발자들이 jsx 파일이 React로 작성된 것임을 알 수 있음.
